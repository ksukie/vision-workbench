# 训练后模型加载与接口说明

[English](../en/trained_model_loading_and_api_README.md) | [返回 YOLO26 训练](./YOLO26训练.md)

这份说明补充 YOLO26 训练后的模型放置、下拉框扫描规则，以及常用 Python API。项目安装和首次运行仍以根目录 README 的“快速开始”为准；二次开发者可以继续看 [二次开发指南](../../二次开发指南.md)。

## 接口层级

项目按功能包拆分，每个功能包大致遵循同一结构：

```text
src/<功能包>/api/             用户优先使用的公开 Python 接口
src/<功能包>/application/     工作流编排，例如加载模型、训练、保存
src/<功能包>/domain/          数据结构和参数对象
src/<功能包>/infrastructure/  文件、模型、相机、第三方库适配
src/<功能包>/processing/      算法或模型调用封装
src/vision_workbench/desktop/  统一 Qt 桌面界面
src/<功能包>/window/          旧 Tkinter 兼容/参考代码，不是推荐入口
```

普通用户优先使用两种入口：

- 图形界面：运行 `vision-workbench`
- Python 调用：从各模块的 `api` 包导入函数

## 常用 Python 接口

目标检测：

```python
from yolo26_detection.api import detect_image

result = detect_image(
    "image.jpg",
    model_path="models/yolo26_models/yolo26n.pt",
)
print(result.detection_count)
```

分割：

```python
from yolo26_segmentation.api import segment_image
from yolo26_segmentation.domain import SegmentationSettings

result = segment_image(
    "image.jpg",
    model_path="models/yolo26_segmentation_models/yolo26n-seg.pt",
    settings=SegmentationSettings(task="segment"),
)
print(result.item_count)
```

训练数据校验：

```python
from yolo26_training.api import validate_dataset, list_models

report = validate_dataset("path/to/data.yaml", task="detect")
print(report.to_text())
print(list_models(task="detect"))
```

图像分类：

```python
from image_classification.api import predict_with_pretrained

result = predict_with_pretrained("resnet18", "image.jpg")
print(result.predictions[:3])
```

## YOLO 训练后放哪里

YOLO 训练完成后，权重通常在：

```text
runs/yolo26_training/<run_name>/weights/best.pt
runs/yolo26_training/<run_name>/weights/last.pt
```

推荐使用 `best.pt`。为了让检测、分割或训练页面的下拉框扫描到它，请复制并重命名到对应目录，然后点击页面里的刷新按钮。

| 训练任务 | 推荐放置位置 | 命名建议 | 会出现在哪些页面 |
| --- | --- | --- | --- |
| `detect` | `models/yolo26_models/custom/` | `my-det.pt` | YOLO 检测、模型训练 `detect` |
| `segment` | `models/yolo26_segmentation_models/custom/` | `my-seg.pt` | YOLO 分割、模型训练 `segment` |
| `semantic` | `models/yolo26_segmentation_models/custom/` | `my-sem.pt` | YOLO 分割、模型训练 `semantic` |

也可以放到用户目录，适合不想改项目文件夹的情况：

```text
~/.vision_workbench/models/yolo26_models/
```

注意：

- 检测模型不要带 `-seg` 或 `-sem` 后缀。
- 实例分割建议带 `-seg` 后缀。
- 语义分割建议带 `-sem` 后缀。
- 不完整或损坏的 `.pt` 文件会被跳过，不会作为可用模型显示。

Windows 示例：

```powershell
New-Item -ItemType Directory -Force models\yolo26_models\custom
Copy-Item runs\yolo26_training\my_run\weights\best.pt models\yolo26_models\custom\my-det.pt
```

## 图像分类训练后放哪里

图像分类训练输出通常在：

```text
runs/image_classification/<run_name>/best.pt
```

GUI 训练完成后会把一份副本放到：

```text
models/image_classification_models/custom/
```

分类页面的自定义模型预测也可以直接点击“选择本地模型文件”加载任意 `.pt` 或 `.pth` checkpoint。

## 远端模型目录

如果维护远端模型 manifest，可以设置：

```powershell
$env:VISION_WORKBENCH_MODEL_MANIFEST_URL="https://example.com/model_manifest.json"
```

用户点击 YOLO 页面里的刷新按钮后，程序会更新本地缓存：

```text
~/.vision_workbench/cache/model_manifest.json
```

没网时仍会使用缓存和内置默认模型清单。
