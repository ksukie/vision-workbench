# 摄像头与视频排查

[索引](./问题排查指南.md) | [English](../en/camera-and-video.md)

本页覆盖摄像头发现、打开失败、读帧失败、截图、录屏、设备权限和设备占用问题。

## 扫描失败或找不到摄像头

检查：

- 摄像头是否连接。
- 是否有其他软件正在占用摄像头。
- 操作系统隐私设置是否允许访问摄像头。
- 重新点击 Refresh Cameras。

Windows 上可以先用系统“相机”应用测试。如果系统应用也打不开，优先处理系统或驱动问题。

## 打开失败

常见表现：

- `Cannot open Camera`
- `Camera open failed`

从下拉框换一个摄像头 route 试试。USB 摄像头可能暴露多个后端，其中一个失败不代表全部失败。

## 读帧失败

常见表现：

- `Camera frame read failed`
- 预览打开后又停止。

降低请求分辨率，或关闭占用摄像头的软件。YOLO 工作流中建议先停止检测，再重新打开摄像头。

## 截图或录屏失败

确认输出目录可写。先尝试 `.png` 截图和 `.mp4` 录屏。如果 MP4 失败，可以改试 AVI，区分是编码器问题还是摄像头问题。

## 预览慢或不稳定

YOLO 检测和分割对 CPU 压力较大。可以降低 image size、使用更小模型，或录屏前停止实时推理。

## 应急清理

如果异常退出后仍有摄像头或 GUI 进程残留，请查看 [应急清理](./应急清理.md)。
