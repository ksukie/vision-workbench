# 环境问题排查

[索引](./问题排查指南.md) | [English](../en/environment.md)

本页覆盖 Python、conda、editable install、统一 Qt 入口、PySide6 启动、路径编码和基础依赖安装问题。

## 命令找不到

常见表现：

- `vision-workbench` 不是可识别命令。
- 旧模块 GUI 命令，例如 `yolo26-detection-workbench`，不再可用。

先检查：

```bash
conda activate vision-workbench
python -m pip install -e .
python -m pip show vision-workbench
```

Vision Workbench 对外只提供一个公开 GUI 命令：

```bash
vision-workbench
```

如果入口命令仍然找不到，可以直接运行 Qt 模块查看真实错误：

```bash
python -m vision_workbench.desktop.app
```

## 导入失败

常见表现：

- `ModuleNotFoundError`
- 从仓库运行 GUI 时无法导入本项目包。

检查：

```bash
cd path/to/vision-workbench
python -c "import cv_basics, vision_workbench; print('ok')"
python -m pip install -e .
```

除非脚本明确支持，否则不要从 `src/` 子目录内部启动项目。

## PySide6 启动问题

常见表现：

- GUI 不打开。
- `PySide6` 导入失败。
- Qt platform plugin 报错。
- GUI 命令立刻返回但没有窗口。

请使用 Python 3.10 或更新版本并安装基础依赖：

```bash
conda create -n vision-workbench python=3.11 -y
conda activate vision-workbench
python -m pip install -e .
```

如果在 Anaconda 环境中导入 `PySide6` 时出现 DLL load failed，请避免把 user-site 的 PySide6 混到 base conda Python 中。建议重新创建环境后在激活的环境内安装项目，或先通过 conda-forge 安装 PySide6，再运行 `vision-workbench`。

如果执行 `vision-workbench` 后命令立刻返回、但没有窗口出现，请用模块方式启动查看真实错误：

```bash
python -m vision_workbench.desktop.app
```

GUI 启动器也会把启动失败日志写到 `%TEMP%\vision-workbench-startup.log`。当 Windows 隐藏 GUI-script 异常时，优先查看这个文件。

## Legacy Tkinter 模块

旧 `*/window/` Tkinter 模块只作为兼容/参考源码保留，不再是公开 GUI 入口。手动运行旧模块遇到 `_tkinter` 错误时，使用带 Tk/Tcl 支持的 Python，或回到统一 Qt 入口：

```bash
vision-workbench
```

## 路径和编码

排查文件问题时优先使用绝对路径。如果路径里有中文、空格或特殊符号，先把项目或数据复制到较短的纯英文路径，例如 `C:\work\vision-workbench`，确认问题是否与路径解析有关。

## 基础依赖安装失败

执行：

```bash
python -m pip install -r requirements.txt
python -m pip install -e .
```

如果镜像源临时不可用，可改用 PyPI 官方源重试：

```bash
python -m pip install -e . --index-url https://pypi.org/simple
```
