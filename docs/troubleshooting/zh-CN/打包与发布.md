# 打包与发布排查

[索引](./问题排查指南.md) | [发布策略](../../legal/发布策略.md) | [English](../en/packaging-and-release.md)

本页覆盖运行版本身份、更新失败、wheel 构建、`dist/`、发布资产和超过 100 MB 的模型文件。

## 程序版本与 `pip show` 不一致

editable 模式下，`pip show vision-workbench` 展示的是上一次 editable 安装生成的发行元数据，而启动器导入的是当前工作区源码。切换分支或修改 `pyproject.toml` 后，两者可能不同，但这不代表启动器仍在运行旧代码。实际运行身份以程序内“版本信息”页面为准。确实需要同步 pip 元数据时，先关闭程序，再在仓库根目录仅刷新 editable 注册：

```bash
python -m pip install --no-deps --force-reinstall --editable .
python -m pip show vision-workbench
```

正式 wheel 安装不使用这条命令修复，应重新安装版本匹配的官方 wheel。

## 检查更新或一键更新失败

离线、GitHub 限流、发布元数据无效、缺少兼容资产、缺少 SHA-256、资产过大或运行依赖契约变化时，都不能显示为“已是最新版本”。可以稍后重新检查，或从版本页面打开官方 Release。若一键安装仍不可用，必须按照该 Release 的说明手动安装，不能绕过哈希或依赖契约门禁。

更新器会对下载资产进行两次复核，并只在 Qt 退出后应用。Python 更新还会在重启前复核已安装 wheel 的身份；Windows EXE 会在替换前自检，并在同目录保留上一版程序。Windows 下可查看 `%LOCALAPPDATA%\VisionWorkbench\updates\<version>\update.log` 获取准确错误。确认所有 Vision Workbench 进程已退出后，可以安全删除失败残留的 `.part` 文件。

## 构建失败

使用：

```bash
python -m pip install build
python -m build
```

如果隔离构建环境无法下载依赖，可以复用当前环境：

```bash
python -m build --no-isolation
```

## dist 目录

`dist/` 是本地构建产物目录，不得提交到源码仓库。

## 发布资产检查失败

发布检查会阻止大模型文件进入 Git：

```bash
python scripts/check_release_assets.py
```

如果模型文件超过 100 MB，请改为本地下载说明、Git LFS 或 GitHub Release assets，再发布。

## Wheel 不是完整运行环境

基础 wheel 会安装第一方 Python 包、必要资源和入口命令，但它不是 Git 仓库的完整副本。测试、开发脚本、完整第三方源码、大型模型权重和可选深度学习依赖可以不包含在 wheel 中。

需要完整项目源码时，请使用与版本对应的 Git tag 或源码归档。需要模型时，请按照模块文档使用内置下载入口、本地模型目录或 GitHub Release Assets。

Python sdist 用于构建基础 Python 包，同样不应被视为完整 Git 仓库源码。各发布物的正式边界见[发布策略](../../legal/发布策略.md)。

Windows 基础 EXE 也有明确边界：包含 Qt 主界面以及基础 CV、全景、相机和版本工作流，不包含重型图像分类或 YOLO 运行环境。需要这些可选工作流时，应使用对应页面提供的源码环境入口。
