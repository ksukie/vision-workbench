# 旧版界面维护策略

[English](./legacy-gui-policy.md) | [文档中心](./README.md) | [返回项目 README](../README.zh-CN.md)

Vision Workbench 的正式图形界面是 `vision-workbench` 命令启动的 PySide6 / Qt 桌面应用。`src/vision_workbench/desktop/` 及其 `pages/` 目录是新增和维护用户界面功能的正式位置。

## 旧版窗口的定位

各功能包中的 `window/` 目录保存早期 Tkinter 界面代码，仅用于兼容现有调用和辅助理解历史实现。它们不是当前推荐的用户入口，也不代表与 Qt 界面具有相同的功能范围。

## 维护边界

- 新的界面功能和交互调整只添加到统一 Qt 桌面应用。
- 模块业务能力应继续通过 `api/`、`application/`、`processing/` 或 `infrastructure/` 提供，不应只实现于旧版窗口。
- 除兼容性维护外，不为 `window/` 增加新的公开入口。
- 用户文档不推荐直接运行旧版窗口；排查历史入口时可以说明其兼容性质。
- 如果未来移除旧版入口，应在 `CHANGELOG.md` 中提前记录影响范围和替代入口。

## 支持说明

项目自动化测试和发布前人工检查以统一 Qt 桌面应用为准。旧版窗口只维持有限兼容，不承诺与 Qt 页面同步新增能力或保持完全一致的操作体验。
